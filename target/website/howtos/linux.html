该操作指导您如何在Linux上使用OmegaT。专门针对 Linux 新手的用户。

Java

OmegaT并非在所有Java版本上都能运转地令人满意。如果您的系统已经安装了Java（很可能如本案例所示），并且您准备使用该现有的全系统Java安装来与OmegaT协同，那么需要首先检查其是否适合。为此，打开一个终端（也就是控制台或命令行窗口；它的通用程序是（KDE）和gnome-terminal（GNOME）），然后输入：

java—版本

该系统会输出已安装在系统层面的java版本的细节信息。

Oracle（以前称为Sun）Java版本：在 OmegaT文档中(readme.txt)查找系统的要求以获得Oracle Java版本所要求的准确信息（也就是您的Java版本是否足够新）。

第三方Java版本：直到最近，OmegaT只能在Oracle/Sun自己的Java上运行。OmegaT现在将运行在由某些Linux发行版提供的OpenJDK Java实现上，但某些功能（如全套命令行选项）可能不可用；可能提供不同的字型集；已经报错了一些小故障，如窗口显示和分段功能。

如果您不确定您的Java版本是否适合，您可以：

  • 仅仅安装一个不带Java的OmegaT版本，观察它是否能正常运转。如果不能，您仍可以在系统层面安装本地或其他不同版本的Java。
  • 在系统层面安装第二个非默认版本的Java，并明确使用此版本启动OmegaT。
  • 安装包含Java的OmegaT版本。

如果您确定在系统级安装的Java版本与OmegaT不兼容，并且您希望将此Java版本替换为Oracle Java版本，请首先检查Oracle软件包是否适用于您的Linux发行版（SUSE，Ubuntu等）。如果这样将使得安装更容易。

如果您直接从Oracle获得Oracle版本的Java，安装可能很简单，但您必须手动配置它才能使其成为系统级别的默认版本。这是我们不推荐的，特别是如果您不熟悉Linux。

第三个选择，即在本地级别安装包含Java的OmegaT版本。这始终是“安全”的选项，并且我们建议使用。

选择和下载OmegaT

以下版本的OmegaT适用于Linux：

“标准”，“最新”，“/ trunk”和“Webstart”。建议使用“最新”版本。（有关这些差异的详细信息，请参阅OmegaT网站上的“下载”页面。）

没有/包含JRE，即没有JRE的跨平台，例如 “OmegaT_2.1.7_02_Beta_Without_JRE.zip”和具有JRE的Linux特定版本，例如，“OmegaT_2.1.7_02_Beta_Linux.tar.bz2”。根据您的Java情况选择不带或带有JRE的版本（请参阅上文）。

作为一个开放源代码的应用程序，OmegaT已经被许多Linux发行版包含在他们的仓库中，所以你可以通过这条路线安装它。但请注意：

  • 大多数Linux发行版默认使用OpenJDK版本的Java。在写此文档时，OmegaT将会在Oracle版本的Java中运行得更好（见上文）。
  • 通常提供的OmegaT版本是“标准”版本。通常“最新”版本会显示更多功能。

请注意，在Linux上安装多个版本的OmegaT而没有发生冲突是很容易的。然而它们将始终访问相同的用户配置文件（除非从不同的用户帐户启动）。

安装 OmegaT

Linux 版本的 OmegaT 现在提供了自己的安装脚本 linux-install.sh 。该脚本：

  • 将 OmegaT 程序文件放入 /opt/omegat/OmegaT-<version>；
  • 包括 bash 和 kaptain 启动脚本；
  • 创建从 /usr/local/bin 到启动脚本的链接。

bash 脚本是一个非常基本的文件，可以通过命令行选项手动编辑，以启动和配置 OmegaT 。

kaptain 脚本（或者称为 kaptain 脚本的“语法”）使您能够从一个用户友好的 GUI 中选择启动和配置选项。例如，您可以使用它来选择用户界面语言，为 OmegaT 分配更多内存，或输入 Google 翻译密码。您可以自定义 kaptain 启动脚本，以使您的偏好设置为默认； 只需在文本编辑器中编辑 omegat.kaptn 文件即可。该文件是用来帮助你。然而，为了便于 kaptain 脚本运转，您必须先安装 kaptain 。如果它不在您发行版的存储库中，您可以 从 kaptain.sourceforge.net 下载。

kaptain OmegaT 启动器的屏幕截图

安装脚本提供了更多的便利，但如果您愿意，仍然可以在 Linux 上手动安装 OmegaT 。只需在您选择的文件夹中解压缩存档。在这种情况下，启动脚本不会写入执行路径。

从命令行启动 OmegaT

OmegaT 不一定要从命令行启动，但这样做可能很有用，因为它提供了调试信息。要从命令行启动，请打开一个终端窗口（不需要root权限）。导航到包含文件 OmegaT.jar 的文件夹，例如：

cd /opt/omegat/OmegaT_2.3.4

并用 ENTER 键确认。

然后输入启动命令本身，并用 ENTER 确认。基本的启动命令是：

java -jar OmegaT.jar

该命令区分大小写。

如果您依赖或希望使用未在系统级别安装的 Java 安装程序，则这还不够：您还必须指示您的系统在哪里查找 Java 。如果您已经安装了提供 JRE 的 OmegaT 版本，则提供的启动脚本已经为您包含此路径。如果您已经安装了额外版本的 Java ，但尚未将其作为系统级别的默认版本，则必须在启动命令中包含 Java 二进制文件的完整路径。这将类似 /usr/lib/jvm/java-6-sun-1.6.0.16/jre/bin/java ；在安装期间记录下安装 Java 的位置，以便您可以再次找到它。因此该命令为（在这种情况下）：

/usr/lib/jvm/java-6-sun-1.6.0.16/jre/bin/java -jar OmegaT.jar

在启动脚本中，不要混淆 OmegaT.jar 的路径和 java 命令的路径。如果启动脚本与 OmegaT.jar 不在同一文件夹中，则需要使用 OmegaT.jar 的路径（以及cd命令）。如果您希望使用的 Java 版本不是系统默认的安装版本，则需要使用 java 命令的 路径。

从启动脚本运行 OmegaT

如前所述， OmegaT 提供了一个 bash 启动脚本一旦该脚本被复制或链接到执行路径（如果你使用提供的安装脚本则会自动完成），可以通过点击 Alt + F2 </ i0>然后键入 omegat </ i1>来启动OmegaT >，之后敲击 ENTER 来确认。

默认情况下，bash 启动脚本是可执行的，但是您的系统可能会更改其权限（例如在解压缩存档时），您可能需要再次更改权限使其可执行。

通过用户友好方式启动 OmegaT

为了以更加用户友好的方式启动 OmegaT ，可以将其添加到您的应用程序启动菜单中或者在桌面上设置启动图标。可用的方式很多，并且由于分布不同，实施这些方法的步骤也很多。

Linux 版 OmegaT 的字体样式

Linux版 OmegaT 的字体样式由于以下因素可能会有较大差异：

  • 所使用的 Java 版本（在 Java 的 Sun 和 OpenJDK 版本上，相同的字体可能显示的样式不同）
  • 可用的字体/所选的字体
  • 字体大小
  • 是否启用了抗锯齿功能（这是一个命令行启动选项，但您也可以在kaptain启动脚本中进行设置）

卸载 OmegaT

从系统中删除所有 OmegaT 的内容：

删除 OmegaT 项目文件夹. 项目文件夹可以在您未来继续使用时重新恢复。

删除 OmegaT 建立的所有用户帐户的文件夹： /home/<user>/.omegat。这将会删除所有已创建的用户设置（例如对分段规则的更改），并且如果文件夹被正确删除（不是仅仅移动到回收站）则无法恢复。

删除所有启动脚本并删除桌面上所有OmegaT的自定义条目。

卸载 OmegaT 不会从您的系统中删除任何 OmegaT 项目，除非您将这些项目放入了 OmegaT 程序或配置文件的文件夹中，最好不要将这些文件放在这些位置。
