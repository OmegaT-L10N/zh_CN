系统要求

OmegaT 可在任何已安装或可安装相应 JRE（Java 运行环境）的系统上运行。OmegaT 现在自带 JRE，无需单独获取。OmegaT 已在 Windows（从 98 开始的所有版本）、Micintosh OS X 及 Linux 上成功安装。

Microsoft Office XML 格式的文件可以直接在 OmegaT 中翻译。旧的 Microsoft Word、Excel 和 Powerpoint 格式的文件可以在转换到对等的当前 Microsoft Office 格式后翻译，或者也可在 OpenOffice.org 或 LibreOffice（同样免费）中转换成 Office Open XML 格式后进行翻译。要获取 OpenOffice.org 或 LibreOffice，请点击左方“第三方软件”按钮链接到相应网站。

带/不带 JRE

OmegaT 需要你的电脑上已安装 Java 运行环境（JRE）才能运行。

如果您确信计算机上已安装了合适版本的 JRE，则可以下载不带 JRE 的 OmegaT 版本。下载会更快，安装需要的硬盘空间会更少。在这种情况下，请不要选择默认版本；选择不同的版本（例如标准版或最新版），然后选择标记为“不包含 Java”的版本。

如果您的计算机上已经安装了 JRE 而您又安装带 JRE 的 OmegaT 版本，OmegaT 的 JRE 会单独安装，不会与之冲突。因此，安装带 JRE 的 OmegaT 版本总是安全的。如果您不确定是否具有合适版本的 JRE，请选择具有 JRE 的版本（或默认版本）。

Mac OS X 用户：Mac OS X 上已默认安装有 JRE；因此，不对 Mac OS X 提供带 JRE 的版本。

Linux 用户：大多数 Linux 发行版都提供了 Java 运行环境，但是一些开源实现无法正常地和 OmegaT 一起运行（尽管这可能不会立即出现）。如果您不熟悉 Linux，我们建议您选择带 JRE 的 OmegaT 版本。更有经验的 Linux 用户可能更喜欢在系统级安装 Oracle（之前是 Sun）JRE。

签名/未签名

Mac OS X 有一个叫做 Gatekeeper 的功能，该功能通过增加运行未签字程序的难度来增加安全性。适用于 Mac OS X 的 OmegaT 有签名和未签名两种版本；它们在功能上是完全相同的，但首选签名版本，因为系统会将其识别为由已知开发者发布。

发布签名版本可能会有延迟。着急的用户也可改用未签名版本；如果您在启动应用时遇到问题，请参考如何打开未认证开发者的应用。
